## 视觉 / 交互

### 3. Timeline 缺乏日历感
- **现象：** TimeBlock 区域是纯白背景，没有像日历 app 那样的色块感觉，不像"block"
- **想法：**
  - 每个 block 加浅色类别背景（如 Core Work 用浅蓝底）
  - 恢复按时长比例的高度，让长 block 视觉上更高（类似日历的时间块）
  - 两者结合后应该能接近 Apple Calendar 的 timeline 感
- **参考：** Apple Calendar day view、Google Calendar

## 分类体系

### 4. Life Admin 细分
- **想法：** 区分「不得不做」（吃饭、洗漱、通勤）和「主动做」（处理杂事、跑腿）
- **可能方案：**
  - 拆成两个类别：`life_necessity`（生活必需）+ `life_admin`（杂事处理）
  - 或在 Life Admin 内加 subtag / 子分类
- **注意：** 类别太多会增加选择成本，需要权衡

### 11. 跨类别活动归类问题（目的 vs 行为）
- **场景：** 为了维系关系 (Social) 去买礼物 (Life Admin)——该归哪类？
- **本质问题：** 分类是按「行为」还是按「目的」？当两者冲突时用户会犹豫
- **可能方向：**
  - 明确一个原则写进 onboarding（如"按主要目的归类"）
  - 或支持多标签 / 主次分类
  - 短期审计建议：按直觉选，description 里补充上下文，事后分析时再判断

### 13. Leisure vs Rest 边界模糊
- **场景：** 玩手机算 Leisure 还是 Rest？刷小红书、看美卡论坛、回消息，内容不同性质也不同
- **思考：**
  - Rest = 被动恢复（睡觉、发呆、冥想）
  - Leisure = 主动娱乐（刷社交媒体、游戏、看剧）
  - 但"躺着刷手机"到底是 rest 还是 leisure？
- **参考：** iPhone Screen Time 的分类方式——按 app 类型自动归类（社交、娱乐、效率工具）
- **可能方向：** 在 description 里记录具体 app/内容，事后分析时区分

## 用户引导 / UX

### 5. 数据安全感 + 新手引导
- **痛点：**
  - 用户会担心数据丢失（localStorage 确实不是最可靠的）
  - 刚开始用时不清楚数据存在 local
  - 8 个分类没有解释，用户不确定某件事该归到哪个类别
- **改进方向：**
  - 首次使用 onboarding：简短说明数据存储方式 + 定期导出提醒
  - 每个分类加一个 `ⓘ` 小图标，点击展开示例（如 Core Work: "投简历、改作品集、面试准备"）
  - 或做一次性的分类解释 tooltip，看过就不再显示
  - 考虑自动定期导出到剪贴板/下载，降低数据丢失焦虑

### 6. 中文支持
- **现状：** UI 全英文，description 可以输入中文但界面不是中文
- **改进方向：**
  - 完整的 i18n 支持（中/英切换）
  - 或者至少把关键 UI 文案做成可配置的
  - 分类名称的中文翻译

### 10. 审计模式应更鼓励输入细节
- **现状：** 当前模式和长期追踪没什么区别，description 是可选的、容易跳过
- **审计的核心价值：** 短期高颗粒度记录，细节越多事后分析越有价值
- **可能方向：**
  - 停止 timer 时弹出 prompt 鼓励补充描述（轻量，不强制）
  - 显示"本条无描述"的视觉提示，温和地提醒
  - 在日总结中统计"有描述的 entries 占比"，让用户意识到
  - 考虑 quick-note 模板（预设常见描述，一键选择）

## 记录能力增强

### 7. 更丰富的笔记 / 语音输入
- **现状：** description 是单行短文本，面向简短标注
- **想法：**
  - 支持多行 notes（点击展开一个更大的输入区域）
  - 支持语音输入转文字（利用浏览器 Web Speech API），快速记录当时的想法
  - 短期审计场景下，记录"在做什么"和"当时在想什么"同样有价值
- **实现考虑：**
  - notes 可以放在 EditSheet 里作为可选的长文本字段
  - 语音输入需要麦克风权限，Safari webapp 模式下兼容性需确认

## 健康 / 工作节奏

### 8. 休息提醒与 Flow 保护
- **矛盾：** 持续工作期间应该适当休息（健康），但提醒不能打破 flow state
- **可能的设计方向：**
  - **静默记录 + 事后回顾：** 不在工作中弹提醒，而是在日总结中标注"连续工作 3h 未休息"，让用户自己调整
  - **温和的视觉提示：** 类似现在的 ReminderBanner 但更轻量——比如 timer 颜色渐变（绿→黄→橙），不打断但有视觉暗示
  - **主动休息按钮：** 在 QuickEntry 区域加一个小的"休息 5min"快捷按钮，让休息也变得 frictionless
  - **总结中加健康指标：** "今天最长连续工作 X 小时"、"建议每 90 分钟休息一次"
- **关键原则：** 绝不用弹窗/声音打断，只用被动的视觉信息

## 方法论 / 使用心得

### 14. 碎片时间是否该记录
- **场景：** 看了下手机回了条消息（2 分钟），要切到 Social 再切回来吗？
- **思考：**
  - 太碎的切换增加记录负担，反而降低主任务的 flow
  - 但审计的目的就是发现碎片化问题——如果忽略就看不到
- **可能的原则：**
  - < 2 分钟的中断：不切换，在 description 里备注"中间回了消息"
  - 2-5 分钟：视情况，如果打断了 flow 就记录
  - \> 5 分钟：应该切换
  - 或者：在日总结中提供"碎片化指数"（切换次数 / 总时长），让数据说话

### 16. Plan/规划活动的分类归属
- **场景：** 做计划、规划工作方式、整理思路——算 Life Admin 还是 Exploration？
- **思考：**
  - Life Admin 偏「维持性事务」（处理杂事、跑腿、日常维护）
  - Exploration 偏「主动探索和学习」
  - 规划工作本身带有「战略思考」性质，更接近 Exploration？
  - 但如果是日常 to-do 整理，又更像 Life Admin
- **可能方向：** 按内容深度区分——浅层整理归 Life Admin，深度思考/策略规划归 Exploration；或者在 description 里标注，事后分析时判断

### 17. 能量标记：开始时 vs 结束时？
- **问题：** 现在的能量标记（⬆️/⬇️）到底标的是 session 开始时的状态还是结束时的？
- **差异很大：**
  - 开始时标 = 记录「带着什么能量状态进入」→ 分析什么时段适合做什么
  - 结束时标 = 记录「做完之后能量如何」→ 分析什么活动消耗/恢复能量
  - 两者对数据分析的意义完全不同
- **可能方向：**
  - 明确定义为其中一种，在 UI 上给文案提示（如「当前能量状态」vs「做完后感觉」）
  - 或支持两个标记（开始能量 + 结束能量），但会增加输入负担
  - 短期审计建议：先统一为一种（建议「开始时」，因为更容易即时标记），description 里补充结束感受

### 18. 倒计时 Timer 功能
- **场景：**
  - 玩手机想控制在 15 分钟内
  - 放空休息想定 5-10 分钟
  - 番茄钟式工作（25 分钟专注）
- **与现有 timer 的关系：** 现有 timer 是正计时（记录已过时间），倒计时是新增功能
- **可能设计：**
  - 在 QuickEntry 长按类别按钮 → 弹出倒计时设置（5/10/15/25/30 min 快捷选择）
  - 或在 timer 显示区域加一个「设定时限」按钮
  - 倒计时结束时：温和提醒（振动 + 视觉变色），不强制停止 timer
  - timer 到期后继续计时但变色，用户自行决定何时停止
- **实现考虑：** 需要在 activeTimer 中增加 `countdown` 字段，UI 显示剩余时间

### 19. Long session 中 break 的记录策略
- **场景：** 工作 2 小时，中间去倒了杯水、上了个厕所（5 分钟），要不要切换记录？
- **两种策略对比：**
  - **忽略 break：** 保持一个 2h 的 Core Work 长 session → 数据显示连续专注力强，但不真实
  - **记录 break：** 2h 被拆成 55min + 5min break + 60min → 数据更真实，但 longest session 指标变短
- **对数据分析的影响：**
  - 忽略：longest session 更长，看起来更好，但掩盖了真实的专注模式
  - 记录：能看到真实的 break 频率和时长，更有分析价值
- **可能方向：**
  - 短期审计建议：< 5 分钟的必要 break（喝水、厕所）可以忽略，在 description 备注
  - 功能层面：考虑「暂停」功能（区别于「停止」），暂停不产生新 entry，恢复后继续同一 session
  - 暂停功能可以和 #18 倒计时结合——暂停时显示 break 时长

### 20. 饭后低能量时段的利用
- **场景：** 饭后需要休息，没法太专注，但又不想完全浪费时间
- **思考：**
  - 这是生理节律问题，不是工具能直接解决的
  - 但工具可以帮助识别和优化这个时段
- **可能方向：**
  - 用数据验证：通过能量标记 + 时段分析，找到自己的低能量时段模式
  - 在日总结中加「时段 × 能量 × 类别」的交叉分析
  - 建议性内容：低能量时段适合做 Life Admin、轻度 Exploration（看文章）、Social
  - 这更多是方法论问题，工具层面可以通过 insights 来辅助

### 15. 持续关注时间导致焦虑
- **现象：** 一直关注时间让用户有点紧张，不敢放松
- **这是审计工具的固有张力：** 观察行为本身会改变行为（霍桑效应）
- **思考方向：**
  - 这可能不完全是坏事——短期审计就是要提高意识
  - 但如果焦虑到影响正常休息就过了
  - **设计层面：** timer 显示可以做得更低调（不显示秒数、缩小字体、或折叠起来只在需要时展开）
  - **引导层面：** 在 onboarding 或首页加一句"记录是为了理解，不是为了评判。Leisure 和 Rest 也是有价值的时间。"
  - **审计结束后：** 考虑"简化模式"，减少 timer 的存在感
